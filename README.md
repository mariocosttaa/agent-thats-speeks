# agent-thats-speeks

Voice assistant monorepo: **Next.js** (React) frontend with Chrome’s **Web Speech API** for fast speech-to-text and browser TTS, and a **Python FastAPI** backend that calls **Gemini 2.5 Flash** with your API key kept on the server.

## Requirements

- **Python** 3.11+
- **Node.js** 20+ (for Next.js)
- **Chromium-based browser** (Chrome or Edge) for reliable speech recognition and synthesis. Safari/Firefox support may be limited.
- **HTTPS or localhost** for microphone access in production.

## Quick start

### 1. API key

Create a [Gemini API key](https://aistudio.google.com/apikey). Copy [`.env.example`](.env.example) to **`.env`** at the repo root and set:

```env
GEMINI_API_KEY=your_key_here
```

Optional: `CORS_ORIGINS=http://localhost:3000` (comma-separated for more origins).

### 2. Backend

```bash
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r backend/requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Run this from the **`backend/`** directory (or `uvicorn backend.main:app` from the repo root with `PYTHONPATH=.`).

### 3. Frontend

```bash
cd frontend
cp .env.example .env.local
# Set NEXT_PUBLIC_API_BASE_URL=http://localhost:8000 if the API is elsewhere
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). Click **Allow microphone**, choose language (pt-PT / pt-BR / en-US / en-GB), TTS voice, and **Live conversation** or **Hold to talk**.

## Speech behaviour notes

- **STT** uses the browser’s speech engine (in Chrome, Google’s). The **microphone dropdown** drives the **level meter** stream; **recognition may still follow the OS default input** where the Web Speech API does not expose device selection.
- **Replies** are generated by **`gemini-2.5-flash`** via the backend; European Portuguese vs English is guided by the selected **input language** and server system prompt.
